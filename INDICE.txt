⚙️ FIREBASE (CONFIGURACIÓN)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

1. 🔑 firebase-config.js
   ├─ Descripción: Credenciales y setup
   ├─ Qué modificar: FIREBASE_CONFIG (línea 10-20)
   ├─ Cómo obtener credenciales:
   │  └─ Firebase Console > Configuración > Web SDK
   ├─ Estructura:
   │  ├─ apiKey
   │  ├─ authDomain
   │  ├─ projectId
   │  ├─ storageBucket
   │  ├─ messagingSenderId
   │  └─ appId
   └─ Importancia: ⭐⭐⭐ CRÍTICO

2. 🛠️ firebase-services.js
   ├─ Descripción: Servicios CRUD reutilizables
   ├─ No modificar: (a menos que sepas qué haces)
   ├─ Contiene:
   │  ├─ AuthService (login, registro)
   │  ├─ UserService (perfiles, config)
   │  ├─ OrderService (pedidos)
   │  └─ ProductService (productos)
   └─ Usado por: admin.html, master.html, index.html

3. 📦 store-config.js
   ├─ Descripción: Carga dinámica de configuración
   ├─ Función: Sincronizar con Firebase
   ├─ No modificar: (código interno)
   ├─ Proporciona:
   │  ├─ STORE_CONFIG (configuración tienda)
   │  ├─ PRODUCTOS (lista dinámica)
   │  └─ CURRENT_USER_ID (usuario actual)
   └─ Usado por: store-script.js

📊 ESTRUCTURA FIRESTORE
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Se crea automáticamente la siguiente estructura:

users/                          (Perfiles de usuarios)
├─ {uid_usuario}/
│  ├─ id: string
│  ├─ nombre: string
│  ├─ email: string
│  ├─ telefono: string (WhatsApp)
│  ├─ nombreTienda: string
│  ├─ colorTema: string (#RRGGBB)
│  ├─ plan: string (free, pro, master)
│  ├─ estado: string (activo, suspendido)
│  ├─ config: map
│  │  ├─ mostrarSorteos: boolean
│  │  ├─ mostrarPromociones: boolean
│  │  └─ comisiones: number
│  └─ timestamp: createdAt, updatedAt

pedidos/                        (Todos los pedidos globales)
├─ {order_id}/
│  ├─ id: string
│  ├─ userId: string (referencia a user)
│  ├─ nombreCliente: string
│  ├─ uidJugador: string
│  ├─ edadCliente: number
│  ├─ metodoPago: string
│  ├─ productos: array
│  │  ├─ [0]: {id, diamantes, precio}
│  │  └─ ...
│  ├─ total: number
│  ├─ estado: string (pendiente, pagado, entregado, cancelado)
│  └─ fecha: timestamp

usuarios_pedidos/               (Índice rápido por usuario)
├─ {uid_usuario}/
│  └─ todos/{order_id}/
│     └─ (copia del pedido para queries rápidas)

productos/                      (Catálogo dinámico)
├─ {product_id}/
│  ├─ diamantes: number
│  ├─ precioBase: number
│  ├─ badge: string
│  ├─ activo: boolean
│  └─ ...

P: ¿Dónde cambio el teléfono WhatsApp?
R: Dos opciones:
   1. En admin.html → Configuración → Teléfono WhatsApp
   2. En firebase-config.js → STORE_CONFIG.telefonoWhatsApp

P: ¿Cómo agrego más productos?
R: En Firestore Console → colección "productos" → Agregar documentos

P: ¿Puedo cambiar los colores?
R: Sí, en admin.html → Configuración → Color Principal

P: ¿Qué pasa si no configuro Firebase?
R: La tienda sigue funcionando sin persistencia (modo anónimo)

P: ¿Puedo usar sin crear usuarios?
R: Sí, la tienda funciona anónima. Pero no se guardan pedidos.

P: ¿Cómo creo un usuario super admin?
R: Ver README_FIREBASE.txt → Paso 6: Crear Usuario Admin

P: ¿Se pueden tener múltiples tiendas?
R: Sí, cada usuario es una tienda con datos aislados.

P: ¿Cuánto cuesta?
R: Firebase free tier: $0 hasta 1 millón documentos/mes

P: ¿Puedo deployer a mi servidor?
R: Sí, todos los archivos son estáticos. Sirve con nginx/apache

P: ¿Dónde veo los datos guardados?
R: Firebase Console → Firestore → Navega las colecciones

📞 SOPORTE Y AYUDA
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Si algo no funciona:

1. Abre navegador → F12 (Consola de desarrollador)
2. Mira si hay errores en rojo
3. Network tab → Ver si request fallan
4. Firebase Console → Revisa logs

Errores comunes:

"Firebase is not defined"
→ firebase-config.js no cargó
→ Verifica orden de scripts en HTML

"Permission denied"
→ Firestore Rules incorrectas
→ O usuario no autenticado

"No puedo hacer login"
→ Usuario no existe en Firebase Console
→ O credenciales incorrectas

"Los productos no aparecen"
→ Sin login: muestra por defecto
→ Con login: busca en Firestore
→ Agrega productos manualmente en BD
